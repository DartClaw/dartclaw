## Project Context

### Project Identity & Stage

**DartClaw** — security-conscious AI agent runtime. Two-layer: Dart host (state, security, API, storage) + harness binaries (Claude Code via JSONL stdin/stdout, Codex via JSON-RPC) spawned and orchestrated directly. Multi-harness is first-class — `HarnessFactory` constructs provider-specific harnesses, `HarnessPool` manages a heterogeneous worker pool. AOT-compiled, zero npm/Node, lean stdlib-first deps. Outpost pattern for foreign-language CLI tools (JSON-IO subprocess, no shared runtime). Lineage: openclaw → nanoclaw → dartclaw.

Stage: **early experimental, soft-published, pre-1.0**. Breaking changes acceptable; no external SDK consumers yet. Workspace versioning is **lockstep** — every publishable `packages/*` + `apps/dartclaw_cli` shares one `version:` value. `dartclawVersion` constant in `packages/dartclaw_server/lib/src/version.dart` is the anchor. Active branch `feat/0.16.5`. Previous tag `v0.16.4` (squash-merge `2f45c84`). Next milestone after 0.16.5 is 0.16.6 (Web UI Stimulus Adoption).

**0.16.5 posture**: Stabilisation & Hardening sprint. Zero new user-facing features. Safety gap fix in alert routing, residual god-file decomposition (post-0.16.4-S45/S46 hotspots: `foreach_iteration_runner.dart`, `context_extractor.dart`, `workflow_executor_helpers.dart`, `config_parser.dart`, `service_wiring.dart`, `cli_workflow_wiring.dart`, `server.dart`), barrel-hygiene formalisation (`dartclaw_workflow` narrowed to ≤35), turn/pool/harness interface extraction down to `dartclaw_core`, 7 orphan sealed events wired to SSE+alert, **13 governance checks installed** (6 L1 fitness + L1 format gate + 6 L2 fitness), `AGENTS.md` + user guide refresh. ### Workspace Layout

```

  dartclaw_models/      # Zero-dep value types (DTOs, sealed hierarchies); bottom of DAG

  dartclaw_config/      # Typed config sections, parser (part files), ConfigNotifier, ConfigWriter
  dartclaw_core/        # sqlite3-FREE runtime: harnesses, channels, EventBus, repo INTERFACES
  dartclaw_storage/     # SQLite-backed repos, FTS5/QMD search, TurnTraceService

  dartclaw_google_chat/ # Chat REST + Pub/Sub + Workspace Events
  dartclaw_testing/     # Curated barrel of fakes/in-memory repos; dev-deps only at consumers

apps/
  dartclaw_cli/         # AOT CLI; ServeCommand boots ServiceWiring.wire()
```

**Per-package conventions/gotchas:**

- **`dartclaw`** — no `lib/src/`. Don't add deps already transitive via core. `umbrella_exports_test.dart` is the contract. Version anchor.
- **`dartclaw_models`** — runtime dep `collection` only. No `path`/`yaml`/`sqlite3`/`dart:io`. Immutable, `copyWith`, explicit `show` in barrel.
- **`dartclaw_security`** — leaf above models (deps: models + logging + path). **Zero EventBus**; verdicts → events via `GuardVerdictCallback` seam (server `SecurityWiring` translates). First-block-wins, fail-closed. `SafeProcess` is the only sanctioned spawner.
- **`dartclaw_config`** — `lib/src/dartclaw_config.dart` is a `part` orchestrator (parsers join part files, not standalone). Channel parsers register on import (call `ensure...Registered()`). `ConfigDelta.hasChanged` is bidirectional-prefix.
- **`dartclaw_core`** — `package:sqlite3` forbidden (arch_check #2). LOC ceiling 13 000 (temporary; S22 closes TD-102 to lower back to 12 000); barrel ≤80 exports. Repo interfaces only here; SQLite impls in storage. `BridgeEvent` (protocol) vs `DartclawEvent` (app) are distinct sealed hierarchies. Harness mutating ops serialized via `_withLock()`.
- **`dartclaw_storage`** — only package besides server+umbrella allowed `package:sqlite3`. `tasks.db` (WAL, authoritative) shared across repos; one `Database` instance.
- **`dartclaw_workflow`** — prod deps capped: config, core, models, security, storage. **Storage dep being removed in S12+S28** (`workflow_task_boundary_test.dart` allowlist empties). Never `Process.run` git directly; inject via `WorkflowGitPort`. Built-in YAMLs in `lib/src/workflow/definitions/` are contract-tested. `dcNativeSkillNames` in `skill_provisioner.dart` is canonical native-skill list.
- **`dartclaw_whatsapp`** — GOWA subprocess; `X-Device-Id` mandatory; webhook ingress in server.
- **`dartclaw_signal`** — signal-cli daemon JSON-RPC + SSE; sealed-sender normalization through `SignalSenderMap.resolve(sourceNumber, sourceUuid)`.
- **`dartclaw_google_chat`** — service-account auth for REST; user-OAuth for Workspace Events; renewal at 75% TTL; `PubSubClient` yields to timer queue every iter.
- **`dartclaw_testing`** — consumers MUST `dev_dependencies` only. Curated barrel `lib/dartclaw_testing.dart`; `public_api_test.dart` pins surface. Naming: `Fake*` boundaries, `InMemory*` repo ports, `Recording*` capture.
- **`dartclaw_server`** — composition root + only package allowed all-channel + storage + sqlite3. `PageRegistry` reserved-prefix guard throws at startup. `TurnManager.reserve` → `TurnRunner.execute` is the canonical turn entry. Workflow-spawned tasks `reviewMode: auto-accept`. Barrel ≤80 exports.
- **`dartclaw_cli`** — `bin/dartclaw.dart` registration-only. Constructor-inject every `dart:io`/factory dep. AOT target: no `dart:mirrors`. `serve` shutdown wires SIGINT/SIGTERM/SIGUSR1 with 10s timeout then forced exit.

### Conventions That Affect FIS

#### Comment policy (drives S37)

**Rationale only, never narration.** Three tiers:

| Tier | Rule |
|------|------|
| Exported public API | Full Effective Dart: one-sentence summary, blank line, body with invariants/throws/preconditions, `[TypeName]` refs, examples if non-trivial |
| Internal (`lib/src/`, not re-exported) | One-sentence summary + only the **non-obvious WHY**. Default: minimal |
| Private (`_`-prefixed) | None unless surprising |

**Anti-rot rules (FIS S37 enforces these via `public_member_api_docs` lint flip + manual sweep):**
- **Drift > absence** — fix or delete wrong/outdated comments on sight.

### Test Strategy

**Four-layer pyramid:**
- **Layer 1 — Unit (~60%)**: Pure behavioural through public APIs. Sociable allowed. Microsecond runtime. Targets: parsers, validators, models, guard chains, state machines.
- **Layer 2 — Integration (~30%)**: Temp dirs + in-memory SQLite + boundary fakes from `dartclaw_testing`. Some suites need `-j 1`.
- **Layer 3 — API/Handler (~8%)**: Real shelf handlers via `handler(Request(...))` — no TCP bind. Real storage + boundary fakes.
- **Layer 4 — Live & E2E (<2%)**: `@Tags(['integration'])` skipped by default. Run with `dart test -t integration`. Provider preset selection via `DARTCLAW_TEST_PROVIDER` env.

**Async patterns:**
- **`pumpEventQueue` from `package:test`** preferred over `Future.delayed(Duration.zero)` (S23 codifies — replaces existing `Duration.zero` patterns).
- `fake_async` mandatory for any timer/loop logic.
- `StreamQueue` (`package:async`) for ordered stream assertions.
- Never real-time waits (`Duration(milliseconds: N)` with N > 0).

## Shared Architectural Decisions

### Arrow-driven contracts (dependent stories)

**1. S01 → S05 — Sealed `DartclawEvent` exhaustive-switch contract**
- WHAT: `AlertClassifier.classifyAlert` and `AlertFormatter._body`/`_details` are `switch (event)` expressions over `sealed DartclawEvent`. Arms returning `null` carry `// NOT_ALERTABLE: <reason>` annotation on the event class. Compiler enforces exhaustiveness — no runtime fitness test.
- PRODUCER: S01.
- CONSUMERS: S05 (adds `LoopDetectedEvent`/`EmergencyStopEvent`/`AdvisorInsightEvent` arms with severity rules — `stuck`→warning, `concerning`→critical, `on_track`/`diverging`→null+NOT_ALERTABLE; wires `MapIterationCompletedEvent`/`MapStepCompletedEvent`/`CompactionStartingEvent`/`TaskReviewReadyEvent` into SSE, adding null arms with `NOT_ALERTABLE` if non-alertable).
- WHY: Compiler-enforced safety net; S05 cannot land without S01's switch in place.

**2. S01 + S09 → S10 — L1 fitness baseline contract**
- WHAT: 6 fitness test files at `packages/dartclaw_testing/test/fitness/*.dart` + `dart format --set-exit-if-changed packages apps` CI gate (NOT a 7th test file). Allowlists committed plain-text at `packages/dartclaw_testing/test/fitness/allowlist/<test-name>.txt`, each line `pattern  # rationale`. No `alertable_events_test.dart` — S01 supersedes via compiler.
- PRODUCERS: S01 freezes "no runtime exhaustiveness test"; S09 freezes barrel-show baseline (≤35 exports with `show`).
- CONSUMER: S10 (writes the 6 tests + format gate).
- WHY: S10 cannot define `barrel_show_clauses_test.dart` allowlist until S09 settles symbol set.

**3. S10 → S22 — Fitness-test regression net**
- WHAT: S10's `max_file_loc_test.dart`, `package_cycles_test.dart`, `barrel_show_clauses_test.dart` operate as the model-migration regression net. Soft re-export shape from `dartclaw_models.dart` (one-release `@Deprecated`) is option, decision recorded in S22 FIS.
- PRODUCER: S10.
- CONSUMER: S22.
- WHY: S22 is High-risk public-API move; relies on mechanical checks.

**4. S10 + S11 → S25 — L2 fitness contract**
- WHAT: 6 L2 tests at the same fitness directory. `dependency_direction_test.dart` encodes allowed pkg edges as committed data with rationale; accepts surgical `dartclaw_workflow → dartclaw_security` edge, rejects `dartclaw_workflow → dartclaw_storage` post-S12. `testing_package_deps_test.dart` enforces `dartclaw_testing/pubspec.yaml` lists only core/models/security (+http if needed). `barrel_export_count_test.dart` per-pkg soft caps: core ≤80, config ≤50, workflow ≤35, others ≤25.
- PRODUCERS: S10 (test directory + allowlist conventions); S11 (final post-extraction `dartclaw_testing` deps shape).
- CONSUMER: S25.
- WHY: S25 needs S11's interface set finalised before encoding allowed-edges table.

**5. S13 → S15 — DRY-helper contract**
- WHAT: S13 produces `mergeContextPreservingPrivate(WorkflowRun, WorkflowContext) → Map<String, dynamic>`, `_fireStepCompleted(stepIndex, success, result)` (or `WorkflowRunMutator.recordStepSuccess/Failure/Continuation`), `truncate()` in `dartclaw_core/lib/src/util/string_util.dart` (char-count; `truncateUtf8Bytes` separate), `unwrapCollectionValue(...)`. `YamlTypeSafeReader` in `dartclaw_config`.
- PRODUCER: S13.
- CONSUMER: S15.
- WHY: S15 carries duplication across the split without these.

**6. S22 → S36 — Public-API CHANGELOG break batching**
- WHAT: Single "Breaking API Changes" CHANGELOG section housing both S22 model moves and S36 renames. `getOrCreateMain*` retains `getOrCreate` prefix (factory-with-side-effect). `kDefault*` constants drop `k`. `getDefaultProject`/`getLocalProject` → getters; `ProviderStatusService.getAll`/`.getSummary` → getters; `GowaManager.getLoginQr`/`.getStatus`, `PubsubHealthReporter.getStatus` → renamed.

- CONSUMER: S36.
- WHY: One coherent migration entry per Decisions Log.

Concrete `WorkflowWorktreeBinder` in `dartclaw_server` implements it. Fake at `dartclaw_testing/lib/src/fake_workflow_task_binding_coordinator.dart`. Hydrate-callback signature drops redundant `workflowRunId` parameter.
