# Changelog

## 0.10.0

- **Added**: `dartclaw-prd` skill — extracts PRD creation out of `dartclaw-plan` into its own altitude. Supports the same dual-mode contract (standalone writes `prd.md` per the output-path semantics; Workflow-Step Mode returns `prd` via `contextOutputs`).
- **Changed (breaking)**: `dartclaw-plan` now requires an upstream PRD (`prd.md` in the input directory, or `prd` via `contextInputs` in Workflow-Step Mode). It fails fast with a redirect to `dartclaw-prd` when no PRD is present. It also absorbs the former `dartclaw-spec-plan` workflow (technical research, story classification, parallel FIS creation, cross-cutting review) and now emits `stories + story_specs` together in Workflow-Step Mode. A new `--skip-specs` flag produces the cheap `plan.md`-only pass; specs are generated by default.
- **Removed (breaking)**: `dartclaw-spec-plan` skill — merged into `dartclaw-plan`. All references swept from in-skill files and workflow definitions.
- **Changed (breaking)**: built-in `plan-and-implement.yaml` rewired around `discover-project → prd → review-prd → plan → story-pipeline → plan-review → remediation-loop → update-state`. The `spec-plan` step is gone, `revise-plan` is replaced by a PRD-scoped `review-prd`, and the merged `plan` step emits both `stories` and `story_specs` in one pass. The mirrored test-workspace workflow (`docs/testing/workflows/data/workspace/workflows/plan-and-implement.yaml`) stays byte-aligned with the shipped built-in.
- Migration: if you use `/dartclaw-plan <dir>` on a directory without a `prd.md`, run `/dartclaw-prd` first to synthesize one, then re-run `/dartclaw-plan`. If you invoke `dartclaw-spec-plan` directly, switch to `/dartclaw-plan` (default) or `/dartclaw-plan --skip-specs` for a plan-only pass. Custom workflow YAML referencing `dartclaw-spec-plan` must migrate to the merged `dartclaw-plan` step shape and add a `dartclaw-prd` step upstream.
