# dartclaw-review-doc

Use this skill to review documentation before implementation starts.

## Review Focus
- Completeness of scope, requirements, and edge cases
- Clarity of terms, boundaries, and acceptance criteria
- Technical accuracy of named tools, APIs, or architectural claims
- Risk areas that would block implementation or mislead delivery
- Over-engineering relative to project scale and stage

## Review Discipline
- Read the document in context of the surrounding project.
- Stay proportional to the maturity of the artifact.
- Challenge findings adversarially before reporting them.
- Prefer concise, actionable findings over exhaustive commentary.

## Report Shape
- Executive summary
- Prioritized findings with severity and rationale
- Readiness assessment
- Recommended follow-up actions
